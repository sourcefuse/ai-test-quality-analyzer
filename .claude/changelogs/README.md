## Purpose

Each changelog documents:
- Files created, modified, moved, or deleted
- Commands executed
- Dependencies added/removed/updated
- Issues encountered and their resolutions
- Git commits made
- Next steps and notes

## File Naming Convention

```
YYYY-MM-DD_HH-MM-SS_session-changes.md
```

Example: `2026-01-19_13-09-57_session-changes.md`

## Usage

Run the `/record-changes` command at the end of a Claude Code session to generate a changelog.

## Structure

Each changelog follows this template:
- **Summary** - Brief overview of the session
- **Changes Made** - Detailed list of all changes
- **Commands Executed** - Bash commands run
- **Dependencies/Packages** - Package changes
- **Issues Encountered** - Problems and solutions
- **Commits Made** - Git commits with hashes
- **Next Steps** - Pending tasks
- **Notes** - Additional context
